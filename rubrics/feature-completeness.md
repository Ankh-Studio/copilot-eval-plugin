# Feature Completeness Rubric

Evaluates feature requests for problem clarity, evidence quality, and validation completeness.

## Criteria

### Problem Clarity (30%)
- **Score 4**: Clear, specific problem statement with user impact
- **Score 3**: Good problem definition with minor ambiguities
- **Score 2**: Basic problem description with gaps
- **Score 1**: Vague or unclear problem statement
- **Score 0**: No clear problem definition

### Evidence Quality (25%)
- **Score 4**: Strong evidence with data, research, and user feedback
- **Score 3**: Good evidence supporting the need
- **Score 2**: Basic evidence present
- **Score 1**: Minimal or anecdotal evidence
- **Score 0**: No evidence provided

### Solution Design (25%)
- **Score 4**: Well-thought solution with alternatives considered
- **Score 3**: Good solution design with minor gaps
- **Score 2**: Basic solution proposal
- **Score 1**: Poor or incomplete solution
- **Score 0**: No solution proposed

### Risk Assessment (20%)
- **Score 4**: Comprehensive risk analysis with mitigation strategies
- **Score 3**: Good risk assessment with key considerations
- **Score 2**: Basic risk identification
- **Score 1**: Minimal risk consideration
- **Score 0**: No risk assessment

## Examples

### Score 4.0
```markdown
## Problem Statement
Users struggle to discover advanced features in the dashboard, leading to 23% feature adoption despite 87% user interest.

## Evidence
- User interviews: 15/20 users unaware of advanced features
- Analytics: 23% adoption rate vs 87% interest
- Support tickets: 40% related to feature discovery
- Competitor analysis: 3/5 competitors have better discovery

## Proposed Solution
Interactive onboarding with contextual tooltips and feature tour.

## Risks & Mitigation
- Risk: Onboarding overwhelms new users
- Mitigation: Progressive disclosure with skip options
```

### Score 2.5
```markdown
## Problem Statement
Users can't find advanced features easily.

## Evidence
Some users mentioned this in feedback.

## Proposed Solution
Add tooltips and a tour feature.

## Risks
Might be too complex for new users.
```

### Score 1.0
```markdown
## Problem Statement
Need better features.

## Evidence
Users want more stuff.

## Proposed Solution
Add more features.
```

## Evaluation Notes
- Check problem statement clarity and specificity
- Verify evidence quality and relevance
- Assess solution completeness and alternatives
- Evaluate risk assessment thoroughness
- Consider success metrics and validation approach
